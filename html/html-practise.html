元信息

头部head标签

所有头部元素的容器，包含文档的标题、脚本、样式、meta、base信息 以及其他更多的信息

title（在头部中，这个元素是必需的）、style、base、link、meta、script、noscript

元数据 meta 标签

提供了 HTML 文档的元数据。元数据不会显示在客户端，但是会被浏览器解析

通常用于指定网页的描述，关键词，文件的最后修改时间，作者及其他元数据。可以被使用浏览器（如何显示内容或重新加载页面），搜索引擎（关键词），或其他 Web 服务调用

如果没有提供 name 属性，那么名称/值对中的名称会采用 http-equiv 属性的值

属性：

charset：定义文档的字符编码。
content：定义与 http-equiv 或 name 属性相关的元信息。
http-equiv：值content-type | default-style | refresh，把 content 属性关联到 HTTP 头部。
name：值application-name | author | description | generator | keywords，把 content 属性关联到一个名称。
scheme：值format/URI，HTML5不支持，定义用于翻译 content 属性值的格式。

移动端显示优化：http://www.runoob.com/w3cnote/viewport-deep-understanding.html

base 标签

规定页面上所有链接的默认 URL 和默认目标

一个文档中，最多能使用一个 base 元素。base 标签必须位于 head 元素内部

属性：
href：规定页面中所有相对链接的基准URL
target：_blank | _parent | _self | _top | framename，页面中所有的超链接和表单在何处打开，会被每个链接中的target属性覆盖

basefont 标签

只有IE9和更早版本的IE浏览器支持该标签。应该避免使用该标签，html4已弃用，h5不支持

标签定义文档中所有文本的默认颜色、大小和字体，使用css代替

程序

客户端脚本script标签和noscript标签

定义客户端脚本，两种方式：内嵌式和外部脚本文件引入

如果使用src引入外部脚本，标签中必须是空的，不允许书写其它代码

对于不能使用脚本或者关闭脚本的浏览器可使用noscript标签提示

在html4中type属性是必须的，h5中是可选的

在XHTML中脚本中的内容类型声明为 #PCDATA（代替 CDATA），就是说会对实体进行解析，意味着，应该编码所有特殊的字符，或者把所有内容嵌套在 CDATA 部分中

属性：

async：规定异步执行脚本（仅适用于外部脚本）,h5新增属性
IE10、Firefox、Opera、Chrome 和 Safari 支持 async 属性
async：是异步加载脚本，不影响文档解析；；无该属性默认方式是立即加载脚本，影响文档解析
xhtml需要完整的属性结构key=value，不可以省略只写属性

defer：文档解析完成后加载脚本
xhtml禁止属性最小化

charset：规定在脚本中使用的字符编码（仅适用于外部脚本）

src：规定外部脚本的 URL。
type：规定脚本的 MIME 类型。
xml:space：HTML5 不支持。规定是否保留代码中的空白。

applet，HTML5不再支持。 HTML 4.01 已废弃。 定义嵌入的 applet，使用object代替

embed 标签-h5新增

定义了一个容器，用来嵌入外部应用或者互动程序（插件），主流浏览器都支持

属性：

height：规定嵌入内容的高度
src：规定被嵌入内容的 URL
type：规定嵌入内容的 MIME 类型
width：规定嵌入内容的宽度

object 标签

定义一个嵌入的对象，主流浏览器都支持

允许您规定插入 HTML 文档中的对象的数据和参数，以及可用来显示和操作数据的代码

用于包含对象，比如图像、音频、视频、Java applets、ActiveX、PDF 以及 Flash

目标是用来取代img和applet，由于漏洞和缺少浏览器支持，未能实现

浏览器的对象支持有赖于对象类型。不幸的是，主流浏览器都使用不同的代码来加载相同的对象类型

object 对象提供了解决方案。如果未显示 object 元素，就会执行位于object之间的代码。这种方式，我们能够嵌套多个 object 元素（每个对应一个浏览器）

属性：

align：top | bottom | middle | left | right，H5不支持。HTML4.01 已废弃。 规定 object 元素相对于周围元素的对齐方式
archive：URL，H5 不支持。由空格分隔的指向档案文件的 URL 列表。这些档案文件包含了与对象相关的资源
border：H5不支持。HTML4.01 已废弃。 规定 object 周围的边框宽度
classid：H5不支持。定义嵌入 Windows Registry 中或某个 URL 中的类的 ID 值，此属性可用来指定浏览器中包含的对象的位置，通常是一个 Java 类
codebase：URL，H5不支持。定义在何处可找到对象所需的代码，提供一个基准URL
codetype：MIME_type，H5不支持。通过 classid 属性所引用的代码的 MIME 类型
data：URL，规定对象使用的资源的 URL
declare：declare，H5不支持。定义该对象仅可被声明，但不能被创建或例示，直到该对象得到应用为止
form：form_id，h5新增，规定对象所属的一个或多个表单
height：规定对象的高度
hspace：H5不支持。HTML4.01 已废弃。 规定对象左侧和右侧的空白
name：为对象规定名称
standby：text，H5不支持。定义当对象正在加载时所显示的文本
type：MIME_type，规定 data 属性中规定的数据的 MIME 类型
usemap：#mapname，规定与对象一同使用的客户端图像映射的名称
vspace：HTML5 不支持。HTML 4.01 已废弃。 规定对象的顶部和底部的空白
width：规定对象的宽度

param 标签

此标签可为包含它的 object 或者 applet 标签提供参数

支持大部分主流浏览器。但是 object 定义的文件格式不是所有的浏览器都支持

属性：

name：定义参数的名称（用在脚本中）
type：MIME_type，H5不支持。定义 MIME 类型参数
value：描述参数值
valuetype：data | ref | object，H5 不支持。描述值

样式/节

style标签

定义 HTML 文档的样式信息

若没有scoped属性，则每一个style>标签必须位于头部区域

属性：

media：为样式表规定不同的媒体类型。 scoped：h5新增，如果使用该属性，则样式仅仅应用到 style 元素的父元素及其子元素。 type：text/css，规定样式表的 MIME 类型。

div 标签

用来组合块级元素，结合css用作布局

span 标签

用来对文档中的行内元素进行组合

不定义任何样式情况下，与其它行内元素无差别

header 标签-h5新增

定义文档或者文档的一部分区域的页眉

应该作为介绍内容或者导航链接栏的容器

IE9及其它主流浏览器支持

footer 标签-h5新增

定义文档或者文档的一部分区域的页脚，可以定义多个footer

包含文档创作者的姓名、文档的版权信息、使用条款的链接、联系信息等等

IE9及其它主流浏览器支持

section 标签-h5新增

定义了文档的某个区域。比如章节、头部、底部或者文档的其他区域

IE9及其它主流浏览器支持

article 标签-h5新增

定义独立的内容

定义的内容本身必须是有意义的且必须是独立于文档的其余部分，如：论坛帖子、博客文章、新闻故事、评论

IE9及其它主流浏览器支持

aside 标签-h5新增

标签外的内容

内容应该与附近的内容相关，常用作边栏

IE9及其它主流浏览器支持

details 标签-h5新增

规定了用户可见的或者隐藏的需求的补充细节

用来供用户开启关闭的交互式控件。任何形式的内容都能被放在 details 标签里边，内容对用户是不可见的，除非设置了 open 属性

与 summary 标签配合使用可以为 details 定义标题。标题是可见的，用户点击标题时，会显示出 details

只有 Chrome 和 Safari 6 支持

summary 标签-h5新增

为 details 元素定义一个可见的标题。 当用户点击标题时会显示出详细信息

只有 Chrome 和 Safari 6 支持

dialog 标签-h5新增

定义一个对话框、确认框或窗口

属性open：规定 dialog 元素是有效的，用户可以与它进行交互

只有 Chrome 和 Safari 6 支持

表格

表格所有标签

table：定义一个表格
border属性可用，是否有边框
cellpadding：H5不支持。规定单元边沿与其内容之间的空白
cellspacing：H5不支持。规定单元格之间的空白
frame：H5不支持。规定外侧边框的哪个部分是可见的
rules：H5不支持。规定内侧边框的哪个部分是可见的
summary：H5不支持。规定表格的摘要
width：H5不支持。规定表格的宽度

caption：定义表格标题

th：定义表格中的表头单元格
abbr：H5不支持。 规定表头单元格中内容的缩写版本
align：H5不支持。 规定表头单元格内容的水平对齐方式
axis：H5不支持。 对表头单元格进行分类
char：H5不支持。 规定根据哪个字符来进行内容的对齐
charof：H5不支持。 规定对齐字符的偏移量
colspan：规定表头单元格可横跨的列数
headers：规定与表头单元格相关联的一个或多个表头单元格
rowspan：规定表头单元格可横跨的行数
scope：规定表头单元格是否是行、列、行组或列组的头部
valign：H5不支持。 规定表头单元格内容的垂直排列方式

tr：定义表格中的行
align：H5不支持。定义表格行的内容对齐方式
char：H5不支持。规定根据哪个字符来进行文本对齐
charoff：H5不支持。规定第一个对齐字符的偏移量
valign：H5不支持。规定表

td：定义表格中的单元
colspan：规定单元格可横跨的列数
rowspan：设置单元格可横跨的行数

thead：定义表格中的表头内容

tbody：定义表格中的主体内容

tfoot：定义表格中的表注内容（脚注）

col：定义表格中一个或多个列的属性值
规定了 colgroup 元素内部的每一列的列属性
span：规定 col 元素应该横跨的列数

colgroup：定义表格中供格式化的列组
用于对表格中的列进行组合，以便对其进行格式化
可以向整个列应用样式，而不需要重复为每个单元格或每一行设置样式
